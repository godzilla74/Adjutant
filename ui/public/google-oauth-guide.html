Google OAuth Setup Guide

How to create a Google Cloud OAuth app and get the Client ID and Secret that Adjutant needs to connect Gmail and Google Calendar.

☁️ Step 1 — Create a Google Cloud project
  1. 1

    Go to console.cloud.google.com and sign in with the Google account you want to use.

  2. 2

    At the top of the page, click the project selector dropdown (it may say Select a project). Click New Project, give it a name like Adjutant, and click Create.

  3. 3

    Make sure the new project is selected in the dropdown before continuing.

🔌 Step 2 — Enable the Gmail and Calendar APIs

Enable Gmail

  1. 1

    Go to APIs & Services → Library → Gmail API.

  2. 2

    Click Enable.

Enable Google Calendar

  1. 1

    Go to APIs & Services → Library → Google Calendar API.

  2. 2

    Click Enable.

📋 Step 3 — Configure the OAuth consent screen

Basic info

  1. 1

    In the left sidebar, go to APIs & Services → OAuth consent screen.

  2. 2

    Select External as the user type, then click Create.

  3. 3

    Fill in App name (e.g. Adjutant), User support email (your email), and scroll down to add your email under Developer contact information. Leave everything else blank and click Save.

Add scopes

  1. 1

    In the left sidebar under your app, click Data Access.

  2. 2

    Click Add or remove scopes.

  3. 3

    In the panel that opens, search for and check each of these scopes:

  • https://www.googleapis.com/auth/gmail.send
  • https://www.googleapis.com/auth/gmail.readonly
  • https://www.googleapis.com/auth/calendar
  1. 4

    Click Update to save the scopes.

Add yourself as a test user

  1. 1

    In the left sidebar, click Audience.

  2. 2

    Under Test users, click Add users and enter your own Google email address. Click Save.

Note: Keeping the app in Testing mode is fine for personal use — only email addresses you add as Test Users can connect. You don't need to go through Google's app verification process.
🔑 Step 4 — Create OAuth credentials
  1. 1

    In the left sidebar, go to APIs & Services → Credentials.

  2. 2

    Click + Create Credentials at the top, then select OAuth client ID.

  3. 3

    Under Application type, select Web application.

  4. 4

    Give it a name (e.g. Adjutant Web Client).

  5. 5

    Under Authorized redirect URIs, click + Add URI and enter exactly: http://localhost:8001/api/oauth/callback

  6. 6

    Click Create. A dialog will show your Client ID and Client Secret — copy both.

What to paste into Adjutant

Client ID→paste into the Client ID field in Settings → Google OAuth
Client Secret→paste into the Client Secret field in Settings → Google OAuth
Note: If you close the dialog without copying the Client Secret, you can retrieve it later by going to Credentials, clicking the pencil icon next to your OAuth client, and clicking Download JSON — the secret is in that file.
