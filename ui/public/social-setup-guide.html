Social Platform Setup Guide

Step-by-step instructions for creating developer apps on each platform and finding the credentials Adjutant needs.

𝕏 Twitter / X

Step 1 — Create a Developer App

  1. Go to developer.twitter.com/en/portal/dashboard and sign in with your Twitter account.
  2. Click + Add App (or create a new Project and add an App inside it).
  3. Give your app a name (e.g. My Adjutant) and click Next.
  4. You'll see your API Key and API Secret — you won't need these for OAuth 2.0, but save them anyway.
  5. Click App Settings, then scroll to User authentication settings and click Set up.

Step 2 — Configure OAuth 2.0

  1. Under App permissions, select Read and write.
  2. Under Type of App, select Web App, Automated App or Bot.
  3. Set the Callback URI / Redirect URL to: http://localhost:7079/api/oauth/callback/twitter (replace the port if yours differs).
  4. Set the Website URL to anything (e.g. https://example.com).
  5. Click Save. You'll see your Client ID and Client Secret.

What to paste in Adjutant Settings

Client ID → paste into Client ID field
Client Secret → paste into Client Secret field
Tip: If you don't see OAuth 2.0 credentials, make sure you completed "User authentication settings" — the API Key shown after app creation is for OAuth 1.0a and is different.
🔵 LinkedIn

Step 1 — Create a LinkedIn App

  1. Go to linkedin.com/developers/apps/new and sign in.
  2. Fill in the App name (e.g. My Adjutant), select a LinkedIn Page to associate (create one if needed — a personal page works), upload any logo, agree to the terms, and click Create app.
  3. You'll land on the App dashboard. Click the Auth tab.

Step 2 — Configure OAuth

  1. Under OAuth 2.0 settings → Authorized redirect URLs for your app, click the pencil icon and add: http://localhost:7079/api/oauth/callback/linkedin
  2. Click Update.
  3. Your Client ID and Client Secret are shown at the top of the Auth tab.

Step 3 — Request the right permissions

  1. Click the Products tab.
  2. Find Share on LinkedIn and click Request access. This is usually approved instantly.

What to paste in Adjutant Settings

Client ID → paste into Client ID field
Client Secret → paste into Client Secret field
Tip: The Client Secret is only shown once when the app is first created. If you missed it, go to the Auth tab → Application credentials and click Regenerate.
🌐 Meta (Facebook & Instagram)

Step 1 — Create a Meta App

  1. Go to developers.facebook.com/apps/create and sign in with your Facebook account.
  2. Select Other as the use case, then click Next.
  3. Select Business as the app type, then click Next.
  4. Enter an App name (e.g. My Adjutant). You can leave the Business Account blank. Click Create app.

Step 2 — Find your App ID and App Secret

  1. In the left sidebar, click App settings → Basic.
  2. Your App ID is shown at the top.
  3. Click Show next to App secret, verify your password, and copy the secret.

Step 3 — Add the Facebook Login product

  1. In the left sidebar, click Add product.
  2. Find Facebook Login for Business (or Facebook Login) and click Set up.
  3. Choose Web, enter your site URL (http://localhost:7079), and click Save.
  4. In the Facebook Login settings, add to Valid OAuth Redirect URIs: http://localhost:7079/api/oauth/callback/meta and click Save changes.

What to paste in Adjutant Settings

App ID → paste into App ID field
App Secret → paste into App Secret field
Tip: While your app is in Development mode you can only post to pages and accounts that are listed as Testers or Developers on the app. To post publicly, you'll need to submit the app for review — but for personal use, development mode is fine.
